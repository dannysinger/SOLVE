Cross Browser Testing

Testing your code across multiple browsers can be a pain, but its crucial do do if you want to maintain a good experience for all users.

There are five main desktop browsers:

  1. Chrome
  2. Firefox
  3. Safari
  4. Internet Explorer
  5. Opera

Chrome has a rapid release schedule and it automatically updates, so you usually only have to support the latest version.

Firefox recently switched to a similar schedule, so supporting the latest version is 100% okay.

Safari is pretty aggressive about updating as well, most support the latest two versions of Safari.

Opera has recently abandoned their browser engine and adopted Chromium / Blink, which is the engine behind Chrome. So, testing for Chrome is essentially testing for Opera.

Internet Explorer is the pain in every developer’s ass. There are currently versions 6,7,8,9,10, and 11 in use. Most IE users are stuck on IE at a corporate job and cannot upgrade. While the future of IE looks very good and fast, at the moment we are stuck supporting buggy versions of IE.

Right now, most developers still support IE8, which is considered the last of the buggy browsers to support.

Working with Analytics Tools

One of great tools offered by Google is their Analytics software. Utilizing this tool allows you to track visitors to your page, their location, age and language, often in real time.

This tool is only available when your website is hosted online and visible to the public.

Installing Google Analytics is easy. Visit http://www.google.ca/analytics/, and log in with your existing Google account.

Once logged in, you should be presented with the option to set up a new property, if not, click the Admin button in the top nav bar.

Within the Admin page, click the dropdown within the Accounts section and choose “Create New Account”.

On the next page, fill out all information pertaining to your site, and you will be presented with your Tracking Info.

Meta Tags

Gone are the days that having your site linked to many others had an effect on your Google Ranking. Now, content, structure and relevance play a part into where your website is ranked when someone does a Google Search.

However, we can define under what relevant topics our page falls in as well, what information is shown in a Google Search by utilizing Meta Tags.

<meta name="keywords" content="Drew Minns, Front End Development, Drew, Minns, Toronto Developer, HackerYou, Hacker You">
<meta name="description" content="Designer, Developer and Educator">
<meta name="author" content="Drew Minns">

Meta tags are placed in between the <head> elements on our webpage and are information sent to services like Google and Facebook. The tags utilize attributes such as ‘name’ and ‘content’ to pass information to these services.

The tags above do the following

  • Keywords: A string of keywords that when searched for, trigger my resulting site.
  • Description: The description that shows up under a search result title in Google.
  • Author: A identifier that allows search engines to index all Articles by a common Author together. ie. All articles by CNN

OG Meta Tags

Open Graph Data within Meta Tags are used by many Services, but in particular, Facebook to present data in a sharable format.

Utilizing these tags, you can have a link to your show up in a status or a post with an Image, Description and Title rather than a simple link. Note that these tags use the ‘property’ attribute rather than the ‘name’ attribute.

<meta property="og:title" content="Drew Minns">
<meta property="og:type" content="website">
<meta property="og:url" content="http://drewminns.com">
<meta property="og:image" content="http://drewminns.com/img/profile.jpg">
<meta property="og:site_name" content="Drew Minns">
<meta property="og:description" content="Designer, Developer and Educator">
  • og:title : The Title of your page/article/etc
  • og:type : The type of content. Examples include video:movie, article
  • og:url : The link to the content
  • og:image : The image that will show up as a preview. This image must be hosted somewhere so others can view it.
  • og:site_name : The name of your overall site
  • og:description : The copy that will display below the title and image as a preview of the content, or as a teaser.

There are many more tags available for use that you can read about here http://ogp.me/

In order to test your website for the correct information, and to see a preview of what your link will look like shared on Facebook, use the debug tool https://developers.facebook.com/tools/debug.

Twitter Cards

Twitter offers tags that will embed content within tweets.

<meta name="twitter:card" content="summary">
<meta name="twitter:url" content="http://drewminns.com">
<meta name="twitter:title" content="Drew Minns">
<meta name="twitter:description" content="Designer, Developer and Educator">

These tags act like normal meta tags, however the name attribute holds details pertaining to twitter information such as the link, the url, title and description.

To read more on Twitter cards, view the documentation on the official site https://dev.twitter.com/cards/overview
