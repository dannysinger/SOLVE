16 - Transforms

CSS3 Transforms

Relativity new to CSS, is css transforms. They allow you to transform an element in a variety of ways, including in a 2D or 3D manner.

The CSS property for transforms is simply just transform, however since it is so new, we have to vendor prefix them. For simplicity, we will be using the webkit (chrome, safari, most mobile devices) vendor prefix like so:

.myBox {
	-webkit-transform:rotate(20deg);
}

However, in any production code, it should look like this:

.myBox {
	-webkit-transform:rotate(20deg);
	-ms-transform:rotate(20deg);
	transform:rotate(20deg);
}

Rotate

The most basic and most useful is rotate, which allows us to rotate an object based on a degree.

<style type="text/css">
.myBox {
	width:200px;
	height:200px;
	background:purple;
	-webkit-transform:rotate(20deg);
}
</style>
<div class="myBox"></div>

You can also apply these effects on hover

<style type="text/css">
.myBox2 {
	width:200px;
	height:200px;
	background:red;
}
	.myBox2:hover {
		-webkit-transform:rotate(50deg);
	}
</style>
<div class="myBox2"></div>

And combine them CSS3 transitions:

<style type="text/css">
.myBox3 {
	width:200px;
	height:200px;
	background:#BADA55;
	transition:all 1s;
}
	.myBox3:hover {
		-webkit-transform:rotate(360deg);
	}
</style>
<div class="myBox3"></div>

rotateX and rotateY

We can also rotateX and rotateY to give you a 3D effect:

rotateX

Flip me!
<style>
	.rotate1 {
		width:200px;
		height:200px;
		background:#3498db;
		transition:all 1s;
	}
	.rotate1:hover {
		-webkit-transform:rotateX(180deg);
	}
</style>

rotateY

Flip me!
<style>
	.rotate1 {
		width:200px;
		height:200px;
		background:#2ecc71;
		transition:all 1s;
	}
	.rotate2:hover {
		-webkit-transform:rotateY(180deg);
	}
</style>

You will notice a few things here:

  1. The flip goes back right away and is jerky. This is because then it shrinks, your mouse hovers off.
  2. The animation is anything but 3D

Scale

Scale is pretty straight forward too, it simply scales an element to be larger or smaller. Lets take the same 200px by 200px block.

The value of scale is a multiple of the original element. So 1 would be the original element. 2 would be twice the size.

<style type="text/css">
.myBox4 {
	width:200px;
	height:200px;
	background:orange;
}
	.myBox4:hover {
		-webkit-transform:scale(1.5);
	}
</style>
<div class="myBox4"></div>

As with the last one, we can also animate them. Note: because this is new, you may see some screen flickering on mobile devices or even desktop browsers.

<style type="text/css">
	.ice {
		transition:all 0.5s;
	}
		.ice:hover {
			-webkit-transform: scale(2);
		}

</style>

<img src="http://nicenicejpg.com/200" class="ice">
<img src="http://nicenicejpg.com/200" class="ice">
<img src="http://nicenicejpg.com/200" class="ice">

We can also combine transforms by simply adding a space between each effect:

<style type="text/css">
	.ice2 {
		transition:all 0.5s;
	}
		.ice2:hover {
			-webkit-transform: scale(2) rotate(1000deg);
		}

</style>

scaleX and scaleY

We can also just scale X or Y, not very useful on images but may be useful if you had a situation where you didn’t want to change the width or height.

<style type="text/css">
	.ice3 {
		-webkit-transform:scaleX(2);
	}
</style>
<img src="http://nicenicejpg.com/200" class="ice3">

skewX and skewY

This is where it starts to get a little twisted, we can use skewX and skewY to skew our images a certain degree

<style type="text/css">
.myBox5 {
	width:200px;
	height:200px;
	background:orange;
	-webkit-transform:skewX(20deg);
}
</style>
<div class="myBox5"></div>

Others

There are a few more:

  • translate: Allows you to move the image X and Y pixels/percentages. Not very useful unless you are doing heavy animation.
  • 3D scales, rotation and transforms: Most of the above will (eventually, some now) allow you to do these on a 3d basis. So instead of rotating on just a 2d field, we can rotate based on a Z value.

http://www.eleqtriq.com/2010/05/understanding-css-3d-transforms/ has as great example.

Process

Since this is such a large project, I want to make sure that we approach it properly. Before we start to code, its important to plan out what we want to build. It doesn’t need to be perfect, but a good plan makes developing websites a lot easier.

Let’s take some time to prep our site and answer some questions:

  1. What information / sections will my side include?
    1. Developing around real content isn’t always possible, but it makes things a lot easier. Figure out what content will go onto your site and develop dummy content or even get the real thing.
    2. Home, About, Contact Form, Pictures, videos, tweets, facebook box, sliders you name it.
    3. Think about every part that goes into your site
  2. What will the layout of my site be?
    1. Look at popular sites and get ideas
    2. Sketch out a rough layout of boxes on paper
    3. Keep in mind a flexible layout that will work well with responsive design
  3. What elements will my site include?
    1. How will the navigation work?
    2. Will I do a single page site? Multiple HTML files?
    3. Are there any JavaScript elements I want to start thinking about? a slider? A lightbox? Cool scroll effect?
  4. What assets do I want to include?
    1. Fonts, buttons, icons, pictures
    2. Start to gather this information together
