Introduction to JavaScript and jQuery

Today we are going to take a peek at JavaScript. While we won’t leave today being JavaScript professionals, we should have a pretty good idea of what JavaScript can do, and when we should be using it.

This is an extremely high level view of view of JavaScript. If what we cover today interests you, I encourage you to explore jQuey and JavaScript further. JavaScript is a tool of a intermediate to advanced Front End developer while HTML and CSS are tools of designers and beginner front end developers.

What is JavaScript

JavaScript is the cool cousin of HTML and CSS. It is a programming language used to add a level of interactivity into website, making the website a little more dynamic than possible with just HTML and CSS. Is is also used for animation, website logic and conditionals. Here are a few examples of when you would use JavaScript in a website:

  • Events — What “events” do we have in CSS? We change change things when we :hover an element. There is also the :checked selector for checkboxes, but that is about it. With JavaScript, we can do anything when anything happens. Click, right click, dropdown select, mouse move, window resize, scroll…
  • Changing CSS — JavaScript is able to dynamically apply/ change CSS.
  • Animations — before CSS animations were around, animation on the web was done by quickly changing CSS styles to produce an animation effect
  • Interactivity — Instead of reloading the website everytime we need to show new content, can we dynamically show it? Think about how facebook or twitter show your status update right away
  • AJAX — We are able to go out to our or another server and request informations. Think of loading tweets/new facebook posts without refreshing the page.
  • Conditionals — JavaScript is a full blown programming language, which means that we can do analysis and conditionally execute javascript.

if(tweet.length > 140) { alert("Tweet is too long!"); } If you are interested in getting started with JavaScript, I recommend getting the small book “JavaScript the good parts” which will give you a good overview of JavaScript and programming in general.

What is jQuery then?

JavaScript the language was written in a week for the Netscape browser. Microsoft copied it with their implementation of “jScript”. Needless to say, JavaScript is a really weird language with way more quirks than CSS. Writing cross-browser JavaScript is really hard and extremely prone to bugs.

So, there is a library called jQuery that makes writing JavaScript much much easier. jQuery is written in JavaScript, but we use the jQuery syntax instead of JavaScript. Take changing the background of an element with a class of box to blue:

document.getElementsByClassName('box')[0].style.color = "blue";

vs

$('.box').css('color','blue');

JavaScript or CSS?!?

In the recent years, CSS3 has taken over many things that we used to do with JavaScript. So, you may find yourself in a case where you could use either. CSS is always preferred however if you need older browser support for things like animations and odd/even rows, you may use JavaScript.

Loading JavaScript

Just like CSS, we can load in external JavaScript. Instead of the link tag, we use the script tag. It takes a src attribute and must be closed. 

<script src="myScript.js"></script>

You may sometimes see the type attribute included, it isn’t required:

<script src="myScript.js" type="text/javascript"></script>

Often times, Google will host JavaScript libraries for us. The benefit of this is speed. If I already have the jQuery library loaded for drewminns.com, when they visit hackeryou.com, it will already be cached in the browser.

<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>

Just like css, we can also write JavaScript right inside our page:

<script>
	alert("Hello there!")
</script>

Best case, you would always load JavaScript files right before the closing </body> tag. Loading them in the <head> works too but isn’t as fast.

Loading jQuery and the document ready

To load jQuery, you simply just include the script tag as we have above.

<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>

Then we write a little bit of code called the document ready:

$(function() {
	// Document is ready! All your code goes inside.
	// You only need one of these per page
});

The document ready is absolutely critical to your jQuery working properly. Javascript works by interacting with the website, so when we run JavaScript, the website needs to be 100% loaded before we do anything.

So, by wrapping our code in a document ready, it will wait until the entire website has finished loading before executing. Otherwise it would fire way too early and we would get unexpected results.

Your page only ever needs a single document ready. For now, put all your jQuery code inside of it. When you get into more advanced JavaScript, you will research other loading techniques.

Selectors

jQuery boils down to select somethingg and then do something with it. jQuery uses CSS selectors, so its easy to write our own.

jQuery uses $('') to wrap selectors and make it a ‘Jquery Object’. So our regular old css selectors transferred to jQuery look like this:

h1 → $('h1')

.box → $('.box')

.nav ul li → $('.nav ul li')

Methods

Once we have selected an element, we can then use what we called methods to do something with that selector. There are hundreds of methods we can use here, we will take a look at a few of them. The rest are available on http://api.jquery.com/jQuery/.

Events

Events are a huge part of jQuery. Instead of just doing something on hover, we can bind to any browser even and do something when that happens. The most common one is click. Events look like this:

$('a.clickMe').click(function() {
	alert("You clicked me!");
});

You may also bind to events using the on method, which may be more readable for you.

$('a.clickMe').on('click',function() {
	alert("You clicked me!");
});

You can also bind to multiple events at once. For instance, the iPhone and iPad have a 300ms delay when you tap a button, which causes noticeable lag. Instead we can bind to touchstart which fires as soon as your finger touches the screen:

$('a.clickMe').on('click touchstart',function() {
	alert("You clicked me or tapped!");
});

CSS

While best practice is to add a class and then style it in the css. We can apply CSS directly with .css()

$('h1').css('color','blue');

$('.box').css('width','200px');

and even multiple values. Note the curly braces and lack of trailing comma on the last line

$('.box').css({
	'width','200px',
	'height' : '300px',
	'background' : 'red'
});
